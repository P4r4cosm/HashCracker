
# Описание работы программы
MD5 и SHA1 в моём коде быстрее работают, но долго инициализируются

MD5 и SHA1 считаются на GPU код на питоне, использована библиотека numba для использования cuda.
Алгоритмы соответственно написаны с нуля для расчётов хешей на gpu.
Тестировалось всё на ArchLinux с cuda 13.0 и видеокартой RTX 3070Ti Mobile 130Watt

Argon2 и BCrypt реализованы на RUST, для использования в python нужно скопилировать RUST модули и интегрировать их.

```sh
maturin develop --release ## Находясь в директории rust_cracker
```

При запуске(запускать файл main_bruteforce.py) программы нужно только ввести хеш, программа сама по хешу определит какой это алгоритм.

# Результаты тестирования
В вариантах где не написано время я не стал проверять, т.к. каждый из хешей бы ломался больше 2 часов (**Очень долго**). Очевидно что, что **Очень долго** на порядок меньше чем **Ещё дольше** и т.д.

| Алгоритм               | H/s         | t, лёгкий пароль, с | t, средний пароль, с | t, сложный пароль, с | t, оч. сложный пароль, с |

| **Hashcat MD5 GPU**    | 13.7-14GH/s | ~0,001 с            | 4.29                 | 8.5                  | 120                      |

| **Hashcat BCrypt GPU** | 1820H/s     | 2304.4              | Очень долго          | Ещё дольше           | Невероятно долго         |
| **Мой код MD5 GPU**    | 17GH/s      | ~0,001 с            | 12.29                | 12.23                | 28.56                    |
| **Мой код SHA1 GPU**   | 5.8GH/s     | 1.01                | 1.06                 | 1.88                 | 51                       |
| **Мой код Argon2 CPU** | 30H/s       | Очень долго         | Ещё дольще           | Невероятно долго     | Бесконечно               |
| **Мой код BCrypt CPU** | 305H/s      | 2432.3              | Очень долго          | Ещё дольще           | Невероятно долго         |

_Да, это не ошибка, моя реализация MD5 оказалась быстрее чем hashcat на gpu_. Единственная проблема - мой алгоритм требует больше времени вначале (порядка 10-12 секунд) из-за этого на простых хешах будет проигрых hashcat.
Также сравнение с hashcat так сильно отличается на больших пароля, т.к. brutforce у hashcat идёт видимо в другом порядке и поэтому он банально перебирает больше вариантов и находит ответ позже. То есть мой алгоритм находит условно за 10 из 100 попыток, а hashcat за 50/100, т.к. как-то по-другому составляет пароль для проверки.
